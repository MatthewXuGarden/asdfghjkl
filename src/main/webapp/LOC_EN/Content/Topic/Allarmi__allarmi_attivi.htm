Purpose

Checking site status and printing alarms.

Acknowledging all alarms.

Open the alarm detail page to acknowledge it and block actions or manually force the alarm.

Description

Most recent alarms are at the top of the list

The row color identifies the priority assigned in the setup (see chapter Device setup: alarm variables)

High priority alarm

Medium priority alarm

Low priority alarm

Linked pages

Alarm details (see chapter Alarm details).

 

If the guardianPRO service is disabled, intercepting any internal faults that jeopardize alarm signaling will be impossible. In fact, guardianPRO makes sure the system promptly manages all current alarms and that alarms stop within a certain time limit.

Acknowledge all alarms only applies to those alarms whose priority was enabled for acknowledgement during setup (see chapter Alarm priorities).

Alarm control frequency can be set for each device (see chapter Device setup: alarm variables)

Action

Common used buttons

double-click an alarm

Opening the Alarm details page.

Acknowledge all alarms: all alarms in the list disappear and can be viewed in the cleared alarm list.

Alarm detail: The time the alarm is acknowledged and user name are recorded.

Events: The time all alarms are acknowledged and user name are recorded.

In Field: If alarm conditions occur the next time the field is read, a new alarm is generated.

Notifications: Actions linked to alarms by a rule are not inhibited. Actions must be stopped for each single alarm.
