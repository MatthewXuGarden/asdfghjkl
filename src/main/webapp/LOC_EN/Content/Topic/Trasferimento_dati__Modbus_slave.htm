Data transfer: Modbus slave

Path:

, Data transfer, page Modbus Slave

Access:

Installers

 

Purpose

Lets variable groups recognized by BMS to be set as a Modbus device.

Description

PVPRO variables (Digital, Alarm, Integer and Analog type) are automatically mapped on Modbus data types:

  • coil (Digital and Alarms)

  • register (Integer and Analog).

Coil and Register have two different addresses. Thus it is normal to have the same address for two types of Modbus variables

What to do next

Start ModbusSlave service in Configuration, System page, System. Subsequent settings changes are applied when the service is rebooted from this page.

The reboot key is only enabled if ModbusSlave service is running. See Configuration, System page, System.

Action

Set connection parameters

  • In Connection configuration select the connection mode with BMS. TCP/IP protocol is set by default on port 502.

Add a Modbus device

  • In Device name indicate the name to assign to the variable group to be displayed.

  • In Device address select an address from those available.

  • In Device select device PVPRO, filtering the device list using the Device models menu if necessary.

  • To select variables:

    • Select the variables and click : an incremental address, starting from zero, is automatically assigned to the variable, and appears on section Current configuration

or

    • in Next Coil/Next register indicate any address

    • Select a variable and click : the indicated address is assigned to the variable and appears in section Current configuration

  • Click Add: the set variable group is added to the Modbus device section.

Enable set R/W variable to writing

  • Select Enable write function

  • Click Save: after rebooting the service, BMS can only write on enabled variables (column Read only not selected).
