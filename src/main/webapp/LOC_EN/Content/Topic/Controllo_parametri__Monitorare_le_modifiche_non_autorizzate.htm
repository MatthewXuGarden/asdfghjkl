Parameter control: Monitoring unauthorized changes

When parameter control is running, field values are read at the end of the interval and the Dashboard page is updated. If necessary, the user can use the values read as default values (see Reading default values). Any changes to parameters due to unauthorized changes or restored default values are recorded in the History page. All default value readings are also recorded.

Change controls

At the end of the set interval, parameter control reads field values and compares them with default values.

    1. If changes are found:

      • If the change was entered on a keyboard or made by an unauthorized user, the change is recorded in the History page. The registration is Modification [1]. If notifications are enabled, addressees are notified.

    1. If no changes are found, no action is taken.

Monitoring field values

To display values read at the last control:

Open the Dashboard page:

  1. The current value appears for each parameter [1].

Monitoring changes

To check for any unauthorized changes

Open the History page:

  1. In Device [1] filter registrations.

Restoring default values

To restore default values for an unauthorized parameter change:

Open the History page:

  1. In Device [1] filter registrations.

  2. Click [2] for the parameter to be restored to start value: the value overwrites the current value: a new registration is recorded without generating notifications.

  3. The registration is Restore [1]. No button appears in the Restore [2] column
