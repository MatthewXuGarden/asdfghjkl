How to create supervision activities

Setting up peripherals

  1. Setting up installed peripherals

    • In PlantVisorPRO Shell, Computer management set the peripherals that are installed (see chapter System-install: computer management)

Setting up notification channels used by peripherals and addressees

  1. Setting up notification channels

    • In I/O configuration, for each channel type (fax, text message -SMS-, e-mail, Remoto) set connection modes (see chapter Notifications-channels: setup).

  2. Defining typical recipients for each notification channel

    • For each physical channel (fax, text message -SMS-, e-mail, Remoto) define a list of potential contacts.

    Always include a channel test contact.

Setting relay actions

  • Setting relays to be started

    • In I/O configuration, Relay, set the activation status for all relays and the timer for timed relays (see chapter Notification-channels: relay setup)

Testing notification channels

  • Simulating an action request

    • In I/O configuration, test all set notification channels selecting test contacts (see chapter Notifications-channels: test).

The communications channel used by scheduled activities must always be tested. An activity that used an channel that was not tested is indicated in the Dashboard page in menu Alarm and event management or Scheduled activity management.

Setting time bands

  • Defining calendars for all notifications

    • In Scheduled activity management or in Alarm and event management, define a recurring event calendar (i.e.: fixed holidays) or occasional events (i.e.: mobile holidays). (see chapter Activities: time bands).

    Calendars are valid for all notification types (alarm or scheduled).

Creating scheduled activities

  1. Creating the action

    • In Scheduled activity management, Actions, create a new action (see chapter Activities: notification and alarm actions).

  2. Defining the action

    • In Scheduled activity management, Actions open the action details: action pages open (see chapter Actions: introduction)

    • Open the channel page (i.e.: report) and specify that required (i.e.: select the report)

    • Return to the scheduled activity Actions page.

  3. Setting the action calendar and rules

    • In Scheduled activity management, Time bands indicate the time band when the action can be run (see Activity: time band)

    • In Rules select the time band, add it to the required action and enable it (see Activity: activation rules).

    • In Synoptic check the period graph (see Activities: synoptic)

Setting alarm/event notifications and actions

  1. Defining single actions

    • In Alarm and event management, Actions, define the single action specifying the recipient (see chapter Actions: setup).

  2. Defining alarm conditions

    • In Alarm and event management, Alarm conditions, select whether the condition is true:

      • for any alarm on any device

      • or for one or more alarms on one or more devices

    • (see chapter Notifications: alarm conditions)

    • or Defining event conditions

      • In Alarm and event management, Event conditions, build the logic expression that compares two variables or a variable and a constant

      • When the logic expression is true, an event is triggered (see chapter Notifications: event conditions).

  3. Setting the rule that triggers activities

    • In Alarm and event managements, Rules, select an action, a condition, a time band and assign it a rule.

    • Enable the rule (see chapter Notifications: rules).

  4. (optional) Disable all alarm and event activities from a device

    • In Site configuration, double-click the line, double-click the device, Main page to disable the alarm and event activities (see chapter Device setup: main page)
