Activity: event conditions

Path:

, Alarm and event management, Event conditions page

 

 

 

Purpose

Create a logical expression between to log variables or between a log variable and a constant which generates an action when it occurs.

Examples

Fan status - device 1 AND Fan status device 2

Regulation temperature > Setpoint

Temperature < 13.5

Description

Operations with Boolean variables: AND, OR, =.

Operations with Integer/Analog variables: >, <, =.

What to do first

Set a sampling period suited to the log variables in the condition.

 

This function only checks log variables from physical devices and logical variables from logical devices.

The system considers the sampling times set on the two log variables and checks whether the condition is true at the end of the shortest time. The sampling period is thus added to the delay set in the Rules page. Events whose conditions are no longer true before the end of the delay + shorter sampling period are ignored.
