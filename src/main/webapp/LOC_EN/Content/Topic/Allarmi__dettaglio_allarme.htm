Purpose

To learn the alarm "history", its physical field causes and user actions.

Acknowledging an alarm.

Stopping actions and notifications.

Clearing insignificant alarms.

Description

-

What to do next

If action is taken on an alarm, note the reasons.

Action

Acknowledging an alarm: the alarm disappears from the active alarm list.

Alarm detail

The time and user name are recorded.

Event

The time and user name are recorded.

In field

If alarm conditions occur the next time the field is read, a new alarm is generated.

Stopping actions: the alarm remains in the active alarm list. Any notification actions (i.e.: text messages, remote calls) or field actions (i.e.: relay or variable settings) can only be stopped before they are run. An action is delayed by x minutes to permit the user monitoring the site to intervene.

Alarm detail

The time actions are stopped and user name are recorded.

Event

The time actions are stopped and user name are recorded.

In field

If alarm conditions cease the next time the field is read, the alarm is autonomously cleared Stopping actions only effect the pertinent alarm and not all alarms of the same type.

Forcing alarms clear: the alarm disappears from the active alarm list and appears in the cleared alarm list even if the field conditions that generated the alarm persist. Usually used for insignificant alarms.

Alarm detail

The time the alarm is acknowledged, action stopped, forced clear and user name are recorded.

Event

The time manually stopped and user name are recorded.

In field

If alarm conditions occur the next time the field is read, a new alarm is generated.
