Enabling variables to be monitored on Masters

  1. In Systems select the Masters to be monitored:

  1. Click to save

Setting and naming Slave groups

  1. In Groups enter the group name and use the arrows in Group settings to add or remove Slaves from the group.

  2. Click to add the group to the list.

  1. Click to save groups, click to edit them or click to delete them.

Linking one or more Slave groups to each Master

  1. In Link double click on a Master device and use the arrows to link it to one or more Slave groups (click to add groups and click to delete them):

  1. Click to save settings.

Setting a second supervision line to determine offline status

  1. In Backup Cfg double click a Master variable in the list: parameters to be set appear.

  1. In System offline backup conditions select Offline or select a variable that signals offline and set the value to be compared.

  2. Click to save settings.

Setting safe parameters and the required status in the event of offline

  1. In Cfg. Devices set the value for each Master variable which, if read, causes safe values to be sent to Slaves.

  2. In Slave devices set the safe and running values for each variable:

  1. In Algorithm settings set:

    • the time after which Safe Restore should consider a Master offline
    • the delay between setting running values to each group

  2. In Action to be run when the Master device is offline select the required status in the event of Master offline.

  3. Click to save settings.
