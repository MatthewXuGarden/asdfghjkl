Activity: alarm conditions

Path:

, Alarm and event management, Alarm conditions page

 

 

 

Purpose

Creates a list of alarm variables that generate an action when triggered.

Description

An alarm condition is triggered when:

  • an alarm variable on any device is triggered

  • one or more alarm variables on different devices are triggered

What to do first

Enable device alarm variables (see chapter Device setup: alarm variables).

 

The system checks for alarms every 30 seconds. This interval is added to the delay set in the Rules page. Alarms cleared before the end of the delay +30 seconds are ignored.

Action

Creating an alarm condition

  • Enter the name of the alarm condition (i.e.:"No bank communications")

  • Select the device and alarm variable to be checked

  • Click to add the variable

  • Repeat to add as many variables to be checked as needed

  • Click
