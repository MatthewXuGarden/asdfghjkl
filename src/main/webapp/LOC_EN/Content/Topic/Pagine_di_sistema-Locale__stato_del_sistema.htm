System pages: system status

Path:

,System pages, System page

 

 

 

Purpose

To administrate the system.

Description

The central table column is intended for actions (buttons, selection boxes). Any diagnostics messages appear in the right column.

 

Learn everything there is to know before editing the components on this page.

Engine status

Stop the engine when you want to isolate the field (i.e.: during unit maintenance).

Start the engine when it is stopped.

Restart the engine when field configuration changes (configuration elements added or deleted) or when a rule changes.

Start/stop that part of the server intended for system data collection (device polling, log sampling, etc.).

Rebooting the system

To find new connected peripherals (i.e.: PS2 mouse and keyboard).

Reboots the entire supervision system

Virtual keyboard

Displays a keyboard on the screen.

After peripheral configuration.

Change start page

By default, the Global monitoring page appears after logon. Use this command to import maps from Layout Editor.

The page is valid for all users.

  • Default: Global view of all devices (see chapter Monitoring: main page).

  • Device list: table device (see chapter Device display).

  • Map: main synoptic map (see chapter Synoptic map display).

Login main language

Set the default language to be presented at login.

Set session length

To guarantee secured and controlled system logon.

When the time is up, the user is automatically logged out.

FTP server

To permit access to CAREL service technicians.

Enables local C:\ftp folder sharing to permit access to service technicians.
