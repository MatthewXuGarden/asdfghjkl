Site setup: alarm priorities

Path:

, Site configuration , Alarm safety page

 

 

 

Purpose

Defines alarm management policies. Decide whether alarms with a certain priority should be acknowledge by users with access privileges and decide how long alarms with a certain priority can remain active (not cleared and not acknowledged) without triggering guardianPRO.

Description

-

Action

Common used buttons

Maximum alarm management time

For each alarm priority, set the maximum active alarm time and enable alarm acknowledgement. The alarm must be cleared or acknowledged by the user (If Ack applicable is enabled) within the indicated time; otherwise guardianPRO will signal the alarm as not managed in time.

Ack applicable (acknowledge)

Ack (acknowledge, alarm recognition). If enabled, the operator acknowledges the alarm which is cleared.

Mandatory note on Ack

If enabled, the mandatory default note is added at alarm recognition.

Default note for Ack

If Mandatory note on Ack is enabled, this is the text that will be automatically added and can be edited by the user.

Persistent alarms

If enabled, the cleared alarm is left in the active alarm list until recognized by the user. If disabled, cleared alarms do not appear in the active alarm list.

Restoring default values.
