System pages: site safety status

Path:

, System pages, Site safety status page

 

click from the menu

 

    

 

Purpose

  1. Check alarms, acknowledge them and display alarm or device details (to block or manually force).

  2. Check critical messages from guardianPRO

  3. Check site safety level.

Description

Most recent alarms are at the top of the list

The row color identifies the priority assigned in the setup (see chapter Device setup: alarm variables)

High-high priority alarm

High priority alarm

Medium priority alarm

Low priority alarm

Linked pages

Alarm details (see chapter Alarm details).

Device detail (see chapter Device monitoring: main page)

 

The site is considered safe when the following elements are set:

  • guardianPRO

  • a heartbeat

  • two channels for alarm signals

This page is only displayed by the icon if site safety status is enabled and there are critical messages from guardianPRO.

Action

double-click an alarm

Opening the Alarm details page.

click on the device name

Link to the Main device page.

Other actions

Disable site safety level display

  • To hide site safety status, click Hide this message

The action is recorded as an event.
