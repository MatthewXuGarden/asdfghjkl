Standard site setup procedures

CAREL devices are controlled by PlantVisorPRO Locale. This type of setup considers all factory device settings.

  1. Setting up field devices

  2. Programming devices

  3. Defining users and access profiles

  4. Setting supervision activities

  5. guardianPRO setup

Setting up field devices

  1. Declaring devices connected to each serial port

    • In Site configuration, Add set serial port features.

    If a USB-RS485 converter is used, follow the wizard.

    • Add physical devices assigning them a serial address

      • For CAREL devices, use the Find device function

    • Edit device descriptions

    • See chapter Line details.

  2. Note any changes made

    • In Site configuration, double-click a line, Notes add a descriptive note of the completed operations (see chapter Note page)

  3. (optional) Disabling a device

    • In Site configuration, double-click a line, Edit line, disable those devices that should not trigger alarms (see chapter Line details)

  4. Test field communications

    • After restarting the engine, the list of all field devices appears in Global.

    • Check device status (in line, disconnected or disabled) and address (see chapter Monitoring: main page )

  5. Printing the physical device list

    • In Global, print the device list (see chapter Monitoring: main page)

  6. Printing configuration settings

    • In REPORT, Commissioning report print parameters (see chapter Print commissioning report)

  7. Resetting alarms and events

    • In System pages, System cancel alarms and events generated during setup (see chapter System status)

For further information, see chapter Setup: basic concepts

Programming devices

  • Program device parameters

    • In Global double-click a device, Parameters page: set device parameters (see chapter Setting device parameters)

    • Propagating parameter settings on similar devices

Defining users and access profiles

  1. Customizing login profiles

    • In User configuration, Profiles assign default users (Installer, Service, Users) group logon and editing permissions (see chapter Logon profiles)

  2. Define users and assign them default profiles

    • In User configuration, Users set typical users and assign them a default profile (installer, Service, User) specifying any restrictions for password expiry and maximum number of login attempts (see chapter User configuration).

For further information, see chapter Program users: basic concepts

Setting supervision activities

  1. Create HACCP and log report templates

    • In Periodic print configuration define which reports will be printed at regular intervals (see chapter Periodic report setup).

  2. Follow the Activity creation procedure to define notification actions, field actions, print actions and warnings.

For further information, see chapter Activities basic concepts

guardianPRO setup

  1. Enable control on alarms that remain active

    • In Site setup, alarm safety set the maximum notification time for each alarm priority (see chapter Site setup: alarm priorities).

    • In System pages, guardianPRO enable the alarm control (see chapter System setup guardianPRO)

  2. Enable variable variation checks

    • In System pages, guardianPRO enable probe variation checks and select variables (see chapter System: guardianPRO setup)

  3. Enable notifications on reserved channels

    • Create at least one privileged notification channel in I/O configuration for guardianPRO notifications adding addressees (see chapter Notification channel setup)

    Always include a channel test contact.

    • In System pages, guardianPRO enable notifications and select addressees (see chapter System pages: guardianPRO)

  4. Test the channel

    • In I/O configuration, test all set notification channels selecting test contacts (see chapter Notifications-channels: test).

Restart the engine after changes.

For further information, see chapter guardianPRO: basic concepts
