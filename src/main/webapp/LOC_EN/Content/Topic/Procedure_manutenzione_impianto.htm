Checking PlantVisorPRO Locale via Internet

  1. Connect to the site via Internet

    • Connect to the Internet and open Internet Explorer

    • In the address bar, enter the static IP assigned to PlantVisorPRO Embedded (see chapter Connection modes): the logon page opens

    • Open PlantVisorPRO Locale as if it were on site

For further information, see chapter PVPRO loc: connection modes

Tracing fault causes from the received notification

  1. Check global alarms

    • Notification text should be sufficiently descriptive to indicate faults.

    • In Alarms, Active alarms page, use the notification time to check which alarms were triggered first (see chapter Active alarms)

    • Double-click an alarm to open details and check if for actions by the site user (see chapter Alarm details)

    • In Alarms, Reset alarms page, double click to open cleared alarm details (see chapter Cleared alarms).

  2. Force minor alarms clear

    • In Alarms, double-click a minor alarm and force it clear (see chapter Alarm details

  3. Check global events

    • In Events check the event sequence, starting from the most recent event (probably near the date-time of the first alarm) to the event that caused the alarm (see chapter Events).

  4. If necessary, run a targeted search

    • In Search, find and print alarms and events that meet search criteria (see chapter Alarm search and Event search).

  5. Analyze the behavior of devices in alarm conditions.

    • Click on the alarm row to open the detail page on the device in alarm conditions.

For further information, see chapter Alarms and events: basic concepts

Muting any warnings

  1. Stop supervisor actions

    • In Alarms, double-click an alarm to open details and stop any scheduled supervisor actions (see chapter Alarm details)

  2. (optional) Disabling a device

    • In Site configuration, double-click a line, Edit line, disable those devices that should not trigger alarms (except the off line one) (see chapter Line details)

  3. (optional) Disable all alarm and event activities from a device

    • In Site configuration, double-click the line, double-click the device, Main page to disable the alarm and event activities (see chapter Device setup: main page)

  4. Stop auxiliary relays.

    • If necessary, in Relay management, stop relays excited by supervisor actions, tied to alarm status (see chapter Deactivating relays).

For further information, see chapter Activities basic concepts

Analyzing device and unit status

  1. Analyze the certain device group variable trends

    • In SITE Global, <groupName>, Log graph: display the variables for the concerned devices and check the trend in the period concerned by the event that generated the alarm (see chapter Monitoring: HACCP and log graphs)

  2. Compare different device variable trends

    • In PLANT, Global, Log graph: display the variables for the concerned devices and check the trend in the period concerned by the event that generated the alarm (see chapter Monitoring: HACCP and log graphs)

For further information, see chapter Graphs basic concepts

Device and unit actions

  1. Checking device status and controlling the unit

    • In PLANT, Global, double-click a device, Main page to:

      • check device status,

      • check probe values,

      • set main parameters,

      • control the device (i.e.: clear alarms)

      • control the unit (i.e.: turn on/off)

    • (see chapter Monitoring-device: main page)

  2. Editing device parameters (i.e.: setpoint).

    • In PLANT, Global double-click a device, Parameters page to set all parameters (see chapter Monitoring- device: parameters)

    • If necessary, propagate the same settings on the same parameters for all similar devices

  3. Editing device group parameters (i.e.: setpoint)

    • In PLANT, Global, <groupName>, Parameters page, set a value on several parameters simultaneously (see chapter Monitoring: parameters)

For further information, see chapter Devices: basic concepts

Checking performance indicators

  1. Check how long values are out of range

    • In ENERGY, Performance indicators (KPI) , click the button for the variables to be checked to request a report and check indicators (see chapter Maps: performance indicators)

Adding new devices

  1. Stop the engine

    • In System pages, System, stop the engine

  2. Add the device to the line

    • In Site setup, select the line, click , select the device type, assign an address, edit the device description and click

    • See chapter Line details.

  3. Setting up a device

    • If necessary, customize the device setup by clicking : Device setup pages open.

    • See chapter Device setup.

  4. Connect the device

    • Physically connect the device to the serial line

  5. Restart the engine

    • In System pages, System, restart the engine

  6. Test field communications

    • After restarting the engine, the new device appears in Global (see chapter Monitoring: main page)

The procedure lest you add devices accepting their factory settings. For further customizations, see procedure Customizing standard device configuration.
