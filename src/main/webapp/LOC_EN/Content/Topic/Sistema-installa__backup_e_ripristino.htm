PlantVisorPRO Shell: backup and restore

Path:

PlantVisorPRO Shell, click Backup

 

 

 

Purpose

To launch, schedule or restore a complete backup.

To backup or restore a configuration backup. 

Service personnel backup.

Description

Indicate the folder where the file will be saved.

Configuration backup/restore is similar to the one that can be launched from the System page.. The differences between backing up from PlantVisorPRO Shell and from the one in the System page are:

  • It can only be launched by an operator who logs onto the machine and not via Web

  • It backups the configuration and software

  • Complete backup can be scheduled at periodic intervals

  • Restore can also import backups of previous software versions

 

During backup, the supervisor system is off. Complex site backup (i.e.: 150 devices for one year of operations) may last about 15 minutes. The system automatically reboots when backup is completed.

We highly suggest backup files are stored in remote files.

When restoring a backup file, pay close attention to the backup version and the currently installed version.

In all supervisors updated to version 1.5, backup is scheduled on the first Tuesday of the month at 6 AM by default.

The Unzip backup function is only used for remote service.

Action

Scheduling a complete backup.

  • In Schedule select the frequency or date for backups and the remote target folder.

  • Click Apply

Launching backup

  • In Backup select the remote target folder

  • Click on the type of backup to launch:

    • Backup complete: full machine copy, used to create a machine clone

    • Configuration backup: backups the site and rules without log data.

    • Service backup: reserved to service personnel.

Restoring backup data

  • In Recovery select the backup file

  • Make sure versions match

  • Click Site Recovery to restore a complete backup

    • or click Configuration Recovery to restore a configuration backup
